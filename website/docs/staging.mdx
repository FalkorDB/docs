# FalkorDB

FalkorDB is a low-latency graph database built on sparse adjacency matrix representation. It supports the OpenCypher query language with proprietary extensions and provides native GraphRAG capabilities for AI applications.

The system is designed for development teams working with complex, interconnected data in production environments where query performance and data relationships are critical.
---

## Overview

FalkorDB implements a property graph database with these characteristics:

- **Data Model**: [Property Graph Model](https://github.com/opencypher/openCypher/blob/master/docs/property-graph-model.adoc) with nodes, relationships, and properties
- **Query Language**: [OpenCypher](http://www.opencypher.org/) with FalkorDB-specific extensions
- **Storage**: Sparse adjacency matrix representation for efficient graph traversal
- **Protocols**: Both [RESP](https://redis.io/docs/reference/protocol-spec/) (Redis) and [Bolt](https://en.wikipedia.org/wiki/Bolt_(network_protocol)) protocol support
- **Indexing**: [Full-text search](/cypher/indexing/fulltext-index), [vector similarity](/cypher/indexing/vector-index), and [range indexing](/cypher/indexing/range-index)
- **Architecture**: Multi-tenant with ACID transaction support
- **GraphRAG**: Native integration via [GraphRAG SDK](/genai-tools/graphrag-sdk) for LLM applications

---

## Use Cases

FalkorDB is suitable for applications requiring:

- **Graph Analysis**: Social networks, supply chain modeling, network topology
- **Knowledge Graphs**: Entity relationship modeling, semantic search, data discovery  
- **Recommendation Systems**: Collaborative filtering based on relationship patterns
- **Fraud Detection**: Pattern matching across transaction networks
- **GraphRAG**: Retrieval augmented generation with structured knowledge graphs
- **Master Data Management**: Complex entity relationships and hierarchies

## Example Usage

<Tabs groupId="programming-language">
<TabItem value="python" label="Python">

```python
from falkordb import FalkorDB

# Connect to FalkorDB
db = FalkorDB(host='localhost', port=6379)
g = db.select_graph('social')

# Create nodes and relationships
g.query("""
    CREATE 
      (alice:Person {name: 'Alice', age: 30}),
      (bob:Person {name: 'Bob', age: 25}),
      (alice)-[:KNOWS]->(bob)
""")

# Query with pattern matching
result = g.query("""
    MATCH (p:Person)-[:KNOWS]->(friend)
    WHERE p.name = 'Alice'
    RETURN friend.name, friend.age
""")

for row in result.result_set:
    print(f"{row[0]}, age {row[1]}")
```

import { FalkorDB } from 'falkordb';

const db = await FalkorDB.connect({

<TabItem value="java" label="Java">

```java
import com.falkordb.*;

Driver driver = FalkorDB.driver("localhost", 6379);
Graph graph = driver.graph("social");

graph.query(
    "CREATE (alice:Person {name: 'Alice', age: 30}), " +
    "(bob:Person {name: 'Bob', age: 25}), " +
    "(alice)-[:KNOWS]->(bob)"
);

Map<String, Object> params = Map.of("name", "Alice");
ResultSet result = graph.query(
    "MATCH (p:Person)-[:KNOWS]->(friend) " +
    "WHERE p.name = $name " +
    "RETURN friend.name, friend.age",
    params
);

for (Record record : result) {
    System.out.println(record.getValue("friend.name"));
}
```

<TabItem value="rust" label="Rust">

```rust
use falkordb::{FalkorClientBuilder, FalkorConnectionInfo};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let connection_info: FalkorConnectionInfo = 
        "falkor://127.0.0.1:6379".try_into()?;
    
    let client = FalkorClientBuilder::new_async()
        .with_connection_info(connection_info)
        .build()
        .await?;
    
    let mut graph = client.select_graph("social");
    
    graph.query(r#"
        CREATE 
          (alice:Person {name: 'Alice', age: 30}),
          (bob:Person {name: 'Bob', age: 25}),
          (alice)-[:KNOWS]->(bob)
    "#).execute().await?;
    
    let mut result = graph.query(r#"
        MATCH (p:Person)-[:KNOWS]->(friend)
        WHERE p.name = 'Alice'
        RETURN friend.name, friend.age
    "#).execute().await?;
    
    for node in result.data.by_ref() {
        println!("{:?}", node);
    }
    
    Ok(())
}
```

- Query latency: Sub-10ms for typical graph traversals
- Storage: Sparse matrix reduces memory footprint by 90%+ compared to dense representations
- Concurrency: Multi-tenant architecture with transaction isolation
- Scalability: Horizontal scaling via replication
